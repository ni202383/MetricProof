## 已知设计风险

### 1. Claim 分类误报与漏报

仅凭 LaTeX 上下文识别实验数字存在不确定性。首版以保守分类、可解释证据、用户确认和 IgnoreRecord 降低风险，但仍需要真实论文样例验证阈值。

### 2. Claim ID 碰撞与文件移动

上下文指纹能抵抗普通插行，但相同表格结构、复制段落或文件重命名仍可能产生碰撞或身份变化。首版必须拒绝模糊自动迁移。

### 3. LaTeX 语法复杂度

自定义宏、复杂表头、跨行跨列和动态表格可能超过首版解析能力。需要通过 limitation diagnostic 受控降级，并避免把局部解析结果当成可靠结构。

### 4. 数值单位歧义

实验文件中的 `0.872` 与 `87.2` 可能使用不同尺度。首版依赖显式 Link scale 和指标配置，自动候选只能提示，不能自动确认。

### 5. 派生语义歧义

“3.1% improvement”可能指相对百分比，也可能误写为百分点。首版要求 DerivedLink 显式声明 operation 和输出单位，不能只从自然语言自动决定。

### 6. 表格标记策略差异

部分论文只加粗一个并列最优值，或使用不同的次优格式。默认策略需要在实现前通过样例验证，并保留显式配置而不扩大为通用样式引擎。

### 7. 配置公平性边界

`UNFAIR_COMPARISON` 的价值依赖用户正确选择 `controlled_keys`。相同配置不证明公平，不同配置也不必然无效，因此报告措辞必须持续保持风险提示性质。

### 8. Git 证据完整性

实验文件中声明的 commit、当前 HEAD 和 dirty 工作树可能不一致。Git 证据只能报告可观察状态，不能自动证明实验确实由某个 commit 生成。
